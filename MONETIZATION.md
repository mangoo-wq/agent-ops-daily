# Agent Ops Daily — Monetization Notes

업데이트: 2026-03-15 (KST)

## 원칙
v1은 매출보다 **자산 축적**이 먼저다.
다만 수익화 경로는 초기부터 설계해 둔다.

## 1) 가장 현실적인 초기 수익화
### A. Sponsorship
- weekly digest 상단 스폰서 1칸
- “brought to you by …” 형태
- low-support, no custom delivery

### B. Affiliate / referral
- agent tooling
- hosting / monitoring / logging / prompt tools
- relevant dev/ops products

### C. Featured directory placement
- 향후 에이전트 툴 / 워크플로우 / 플레이북 디렉토리로 확장 시 사용

## 2) 나중에 가능한 확장
### D. Premium report
- 월간 trust/evaluation/authority 리포트
- 단, 사람이 많이 붙으면 우선순위 낮춤

### E. Dataset bundle
- 공개 신호를 정리한 structured export
- 자동 생성 가능하면 검토

## 3) 피해야 할 수익화
- 맞춤 컨설팅
- bespoke 분석 리포트 납품
- 운영 대행
- 실시간 고객지원이 많이 필요한 상품

## 4) 우리가 진짜 원하는 상태
- 콘텐츠 생성 루프는 자동
- 배포는 반자동 또는 자동
- 매출은 광고/스폰서/제휴처럼 low-touch
- 사장님은 방향성만 보고 승인

## 5) 현실 체크
트래픽형 모델은 빠르게 큰 돈 벌기 어렵다.
하지만 서버비/운영비/CS 스트레스가 훨씬 낮고,
사장님 성향에는 더 잘 맞는다.
